# Sync Runbook: Live Setup ↔ Occam's Code Repo

> **Purpose:** Guide an AI agent through syncing changes between a live OpenCode setup and the occams-code public repo. The user points you here when they want to sync.
>
> **This is not automatic.** It requires judgment — personal data must be scrubbed, intentional differences preserved.

## Overview

There are two directions:

- **Live → Repo:** You improved your live setup. Publish to the repo (scrubbed).
- **Repo → Live:** The repo got improvements (PRs, etc.). Backport to your setup (restore personal context).

## File Map

### Files that SYNC (identical content, both directions)

| File | Live location | Repo location | Notes |
|------|--------------|---------------|-------|
| Launcher | `~/.config/opencode/bin/oc` | `bin/oc` | ~351 lines. No model tweaker, no startup health checks |
| Scripts | `~/.config/opencode/scripts/*.py` | `scripts/*.py` | 4 files: wiki-lint.py, project-init.py, repo-ingest.py, detect-project-state.py |
| Commands | `~/.config/opencode/commands/*.md` | `commands/*.md` | 4 files: permissions, preset, remember, wiki |

### Files that SYNC with SCRUBBING (live → repo only)

| File | Live location | Repo location | What to scrub |
|------|--------------|---------------|---------------|
| AGENTS.md | `~/.config/opencode/AGENTS.md` | `AGENTS.md` | See scrubbing checklist below |
| oh-my-opencode-slim.json | `~/.config/opencode/oh-my-opencode-slim.json` | `config/oh-my-opencode-slim.json` | Default preset, fallback chain order |

### Files that COPY directly (live → repo)

| File | Live location | Repo location | Notes |
|------|--------------|---------------|-------|
| opencode.json | `~/.config/opencode/opencode.json` | `config/opencode.json` | Copied as-is (no generation step) |

### Files that DO NOT SYNC

| File | Why | Lives where |
|------|-----|-------------|
| `auth.json` | API keys — NEVER share | Live only |
| `budget.json` | Personal spending data (if used) | Live only |
| `package.json` / `bun.lock` | Auto-generated by npm/bun | Live only |
| `backups/`, `logs/` | Personal session data | Live only |
| Wiki content (`~/wiki/`) | Personal knowledge base | Live only (repo has templates) |
| `install.sh` | Repo-only (for fresh installs) | Repo only |
| `README.md`, `INSTALL.md`, `LICENSE`, `CLAUDE.md` | Repo documentation | Repo only |
| Wiki template files | Clean templates | Repo only |

## Intentional Differences

These differences are **deliberate** — do not "fix" them:

| Aspect | Live Setup | Repo |
|--------|-----------|------|
| Default preset | Your chosen default (e.g., zai-coding-plan) | `balanced` |
| Fallback chain order | Your preference (e.g., subscription-first) | OpenRouter-first (broadest compatibility) |
| Workflow Principles intro | May mention your preset/subscription | Generic (no preset-specific text) |
| Council default | Your preset | `balanced` |
| Providers table order | Your primary first | openrouter first, zai-coding-plan last |
| Auth paths | May have personal paths (e.g., `~/.local/share/opencode/auth.json`) | Generic `auth.json` |
| Presets | 4 (including zai-coding-plan) | 3 (balanced, cheap, premium) — zai-coding-plan is live-only |
| Per-project config | `.opencode/oh-my-opencode-slim.json` with personal overrides | Same file, plugin reads natively, deep-merges |
| Direct provider routing | e.g., `anthropic/claude-sonnet-4-6` (no OpenRouter markup) | `openrouter/anthropic/claude-sonnet-4-6` (works for all users) |

## Scrubbing Checklist

When copying live → repo, check for and remove ALL of these:

### Personal references
- [ ] `/home/<username>/` or `$HOME` with specific paths → replace with generic or remove
- [ ] Your GitHub username / real name / org name
- [ ] Personal project names in examples or defaults
- [ ] Personal auth paths (e.g., `~/.local/share/opencode/auth.json` → `auth.json`)

### Preset/subscription context
- [ ] Lines mentioning your specific preset as "active" → change to `balanced (default)`
- [ ] "subscription-based" / "subscription APIs are slower" / "compensate with parallelism" → remove
- [ ] "concurrency mitigates subscription latency" → simplify to just "parallel"
- [ ] Your provider listed first → reorder to openrouter-first
- [ ] References to `zai-coding-plan` preset → remove (live-only, not in repo config)

### Config-specific
- [ ] `z-ai/*` or `zai-coding-plan/*` model IDs in fallback chains → replace with openrouter equivalents
- [ ] Council default pointing to your preset → point to `balanced`
- [ ] Custom model table in AGENTS.md → replace with pointer to oh-my-opencode-slim.json
- [ ] Per-project overrides in `.opencode/oh-my-opencode-slim.json` → strip personal models
- [ ] Z.ai Vision MCP `Z_AI_API_KEY` value → must be empty string `""` (never share API keys)

### Verification
After scrubbing, run: `grep -rn "personal_username\|/home/\|specific_project_name" repo/` — should return 0 matches.

## Step-by-Step: Live → Repo Sync

### 1. Detect changes
```bash
diff ~/.config/opencode/bin/oc ~/personal/repos/occams-code/bin/oc
diff ~/.config/opencode/commands/ ~/personal/repos/occams-code/commands/
diff ~/.config/opencode/scripts/*.py ~/personal/repos/occams-code/scripts/*.py
diff ~/.config/opencode/AGENTS.md ~/personal/repos/occams-code/AGENTS.md
diff ~/.config/opencode/oh-my-opencode-slim.json ~/personal/repos/occams-code/config/oh-my-opencode-slim.json
diff ~/.config/opencode/opencode.json ~/personal/repos/occams-code/config/opencode.json
```

### 2. Review each diff
For each changed file, ask: **"Is this a personal customization or a general improvement?"**
- General improvement → copy to repo (scrub if needed)
- Personal customization → skip, it's an intentional difference

### 3. Copy and scrub
For files in the "SYNC with SCRUBBING" category:
1. Copy the file
2. Apply the scrubbing checklist
3. Verify no personal data leaked

For files in the "SYNC" category:
1. Copy directly — no scrubbing needed

For `opencode.json`:
1. Copy from live to `config/opencode.json` as-is
2. Check for personal MCP paths — strip or generalize if present

### 4. Verify
```bash
# No personal data in repo
grep -rn "your_username\|/home/your_username\|specific_project" ~/personal/repos/occams-code/

# Repo files are valid
python3 -c "import json; json.load(open('config/oh-my-opencode-slim.json'))"

# Line count reasonable
wc -l AGENTS.md  # Should be ~100 lines
wc -l bin/oc     # Should be ~351 lines
```

### 5. Commit and push
```bash
cd ~/personal/repos/occams-code
git add -A
git diff --cached --stat
# Review the staged changes
git commit -m "Sync: <description of what changed>"
git push
```

## Step-by-Step: Repo → Live Sync

### 1. Detect changes
Same diff commands as above, reversed.

### 2. Copy files
For "SYNC" files: copy directly.
For "SYNC with SCRUBBING" files: copy, then **restore personal context**:
- Re-add your preset as active
- Re-add subscription latency notes
- Re-add personal auth paths
- Re-order providers with your primary first
- Re-add `zai-coding-plan` preset to oh-my-opencode-slim.json

### 3. Verify
```bash
# Live setup still works
~/.config/opencode/bin/oc --help

# Personal bits intact
grep "your_preset.*active\|subscription" ~/.config/opencode/AGENTS.md
```

### 4. Commit to live repo (if tracked)
```bash
cd ~/.config/opencode
git add -A
git commit -m "Sync from occams-code repo: <description>"
```

## Safety Rules

1. **Never copy `auth.json`, `budget.json`, or `mcp-oauth.json`** — secrets
2. **Never copy wiki content** — personal knowledge base
3. **Always verify after sync** — diff both ways to confirm intentional state
4. **When in doubt, ask the user** — don't guess if something is personal or general
5. **Test the live setup after repo→live sync** — run `oc --help` to verify
6. **Check `opencode.json` for personal MCP paths** before copying to repo
