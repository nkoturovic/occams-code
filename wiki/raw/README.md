# Raw Sources (Immutable)

Karpathy wiki pattern: `raw/` is the source of truth. **Immutable** — the LLM reads from here but never modifies anything.

## How to add sources

Just drop files into the right subfolder. No special formatting needed.

- **Web articles** → `articles/` (use Obsidian Web Clipper to convert to markdown)
- **Papers/PDFs** → `papers/`
- **GitHub repos** → `repos/` (use `oc --ingest-repo owner/repo` for automated digests)
- **Documentation** → `docs/`
- **Forum posts/discussions** → `forums/`
- **Images/screenshots** → `assets/`
- **Unsorted** → `_inbox/` (triage later)

## Naming convention

`YYYY-MM-DD_descriptive-slug.md` (e.g. `2026-04-04_karpathy-llm-wiki.md`)

For repos: `YYYY-MM-DD_owner--repo/` (auto-generated by `oc --ingest-repo`)

## What happens after adding

The LLM reads raw sources and compiles knowledge into `wiki/` pages. You can:
1. Ask the agent to ingest a specific source: "Read raw/articles/X and update the wiki"
2. The agent will discover sources when they're relevant to a question

## Rules

- **Never edit files in raw/** — if you find an error, add a correction note, don't modify the source
- **Never delete files in raw/** — the wiki may reference them
- **Structure doesn't matter much** — the subfolder is a hint, the content is what matters
